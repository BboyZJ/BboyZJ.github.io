相册 权限

  • iOS11之前：访问相册（读权限） 和 存储照片到相册（写权限），需要用户授权，需要添加Privacy - Photo Library Usage Description（读权限）和 Privacy - Photo Library Additions Usage Description (写权限)

    • Privacy - Photo Library Usage Description 读权限： App需要更换头像图片，是否允许访问您的相册？
    • Privacy - Photo Library Additions Usage Description 写权限： App需要添加图片到相册，是否允许访问您的相册？

  • iOS11之后：默认开启访问相册权限（读权限），无需用户授权，无需添加

注：适配iOS11之前的还是需要加的

相机 权限

  • iOS11之前：访问相机，需要用户授权，需要添加 Privacy - Camera Usage Description

    • Privacy - Camera Usage Description : App需要扫描二维码或拍照，是否允许访问您的相机？

  • iOS11之后：默认开启访问相机权限，无需用户授权，无需添加

注：适配iOS11之前的还是需要加的

麦克风 权限

访问 麦克风 需要用户授权，否则App会crash：

Privacy - Microphone Usage Description ： App需要您的同意，才能访问麦克风，以便您干什么

判断麦克风是否授权：

当 authStatus == AVAuthorizationStatusNotDetermined 时，代表 权限从未询问过，需调用 requestAccessForMediaType 方法进行弹框询问

定位 权限

  • Privacy - Location When In Use Usage Description ： App需要您的同意，才能在使用期间访问位置，以便您干什么

  • Privacy - Location Always Usage Description ： 需要您的同意，才能始终访问位置，以便您干什么？

判断是否已开启地理位置权限：

询问是否弹窗授权开启定位权限：

当 [CLLocationManager locationServicesEnabled] && [CLLocationManager authorizationStatus] == kCLAuthorizationStatusNotDetermined 时代表从未询问过位置权限，需requestWhenInUseAuthorization方法进行弹框询问

询问结果

弹框询问是否开启使用位置后，通过 CLLocationManagerdelegate 方法 -- (void)locationManager:(CLLocationManager*)manager didChangeAuthorizationStatus:(CLAuthorizationStatus)status 进行回调得知询问的结果 status。
